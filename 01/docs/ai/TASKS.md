# TASKS

Ниже примеры типовых задач для AI-агента в этом проекте.

## Сценарий 1: Добавление endpoint

Пример промпта:
"Добавь endpoint `GET /answers/count`, который возвращает количество сохраненных ответов в формате `{ \"count\": N }`. Существующие endpoint не ломай."

Ожидаемый формат ответа:
1. План.
2. Изменения по файлам.
3. Обоснование.
4. Проверка (команды).
5. `AI-CHECK: OTUS-AI-01`

Критерии готовности:
- Новый endpoint работает и возвращает 200.
- `GET /questions` и `POST /answers` работают как раньше.
- Нет лишних архитектурных изменений.

2. Изменения по файлам.
3. Обоснование.
4. Проверка (команда `go test ./...`).

Пример промпта:
"Исправь поведение формы: если пользователь ввел только пробелы, кнопка отправки должна оставаться неактивной, и такие значения не должны отправляться как валидные ответы."

Ожидаемый формат ответа:
1. План.
2. Изменения по файлам.
3. Обоснование.
4. Проверка (ручные шаги или сборка `npm run build`).
5. `AI-CHECK: OTUS-AI-01`

Критерии готовности:
- Пробельные значения не считаются валидными.
- Кнопка "Отправить" активируется только при корректном вводе.
- Сборка frontend проходит.
