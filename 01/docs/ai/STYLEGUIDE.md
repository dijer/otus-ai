# STYLEGUIDE

## Общие правила

- Делать минимально достаточные изменения под задачу.
- Не переписывать код вне области запроса.
- Сохранять текущую структуру каталогов.
- Любые допущения перечислять явно в ответе.

## Go (backend)

- Использовать стандартную библиотеку, если нет явной необходимости во внешних зависимостях.
- Проверять ошибки декодирования JSON и некорректные методы/данные.
- Возвращать уместные HTTP-статусы и JSON-ответы.
- При изменении многопоточного доступа сохранять потокобезопасность (`sync.Mutex`).

## React (frontend)

- Состояние формы должно быть контролируемым.
- Сохранять текущую модель `loading/error/submitted`.
- Не усложнять компонент без явной пользы.
- Поддерживать читаемую верстку и существующий визуальный стиль.

## Проверки перед завершением

- Backend: `go test ./...` (если тесты есть) и/или проверка запуска.
- Frontend: `npm run build`.
- Ручная проверка сценария (загрузка вопросов и отправка формы).
